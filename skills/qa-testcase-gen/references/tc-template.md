# 테스트 케이스 템플릿

## 마크다운 출력 포맷

```markdown
# [QA] {기능명} 테스트 케이스

- **PRD:** {Confluence 링크}
- **Figma:** {Figma 링크}
- **Jira:** {관련 티켓 링크}
- **작성일:** {날짜}

---

## 1. 기능 테스트

| ID | 시나리오 | 사전조건 | 테스트 단계 | 기대결과 | 우선순위 |
|---|---|---|---|---|---|
| TC-001 | {시나리오 요약} | {필요한 상태/조건} | 1. {단계1}<br>2. {단계2} | {기대되는 동작} | P0/P1/P2 |

## 2. 권한/접근 테스트

| ID | 역할 | 대상 페이지 | 기대 동작 | 우선순위 |
|---|---|---|---|---|
| TC-010 | {역할명} | {페이지명} | 접근 허용 / 구독유도 팝업 | P0/P1 |

## 3. 경계값 테스트

| ID | 항목 | 입력값 | 기대결과 | 우선순위 |
|---|---|---|---|---|
| TC-020 | {제한 항목} | N-1 / N / N+1 | {각각 기대 동작} | P0/P1 |

## 4. UI/UX 테스트

| ID | 화면/컴포넌트 | 검증 항목 | 기대결과 | Figma 참조 |
|---|---|---|---|---|
| TC-030 | {화면명} | 문구/레이아웃/색상 | {Figma와 일치} | {nodeId} |

## 5. 엣지케이스

| ID | 시나리오 | 조건 조합 | 기대결과 | 우선순위 |
|---|---|---|---|---|
| TC-040 | {예외 상황} | {복합 조건} | {기대 동작} | P1/P2 |
```

## 우선순위 기준

| 등급 | 기준 | 예시 |
|---|---|---|
| **P0** | 핵심 비즈니스 로직, 결제, 접근 차단 | 구독 유도 팝업 미노출, 결제 오류 |
| **P1** | 주요 기능 동작, 권한 검증 | 대화 횟수 카운트 오류, 페이지 접근 권한 |
| **P2** | UI 텍스트, 레이아웃, 부가 기능 | 문구 불일치, 정렬 오차 |

## Confluence 출력 포맷

Confluence 페이지 생성 시 마크다운 테이블 대신 ADF(Atlassian Document Format)의 table 사용.
`createConfluencePage`에 `responseContentFormat: "markdown"` 지정.

## Jira 하위 Task 포맷

TC를 Jira 하위 Task로 등록 시:
- **summary:** `[QA] TC-{ID}: {시나리오 요약}`
- **description:** 사전조건 + 테스트 단계 + 기대결과
- **issuetype:** 하위 작업
- **priority:** P0→Highest, P1→High, P2→Medium
