# Figma Implementation Rules

피그마 기반 구현 요청에서 개인 스타일 규칙을 적용한다.

## 사용 시점

-   사용자가 피그마 링크/노드와 함께 UI 구현을 요청할 때
-   디자인 정합성을 높여야 하는 컴포넌트/페이지 작업

## 핵심 원칙

1. 피그마에 적힌 기획 내용을 먼저 확인한다.
2. 구현 전에 기존 컴포넌트가 있는지 확인하고, 있으면 재사용한다.
3. 다른 파일에 동일/유사 디자인이 있는지 찾아 일관성을 유지한다.
4. 컬러, 폰트, 아이콘, 간격을 피그마와 동일하게 맞춘다.
5. 피그마에 없는 디자인/아이콘을 임의로 만들지 않는다.
6. taillwind css 의 leading 사용 금지
7. #000000 컬러값이나 rounded-[8px]을 rounded-lg를 taillwind로 변경 가능하면 하기
8. px 대신 rem 사용

## 실행 절차

1. 피그마 컨텍스트 수집
    - Figma MCP로 노드의 스펙/스크린샷/변수 확인
2. 코드베이스 재사용 탐색
    - 관련 컴포넌트 검색
    - 동일/유사 레이아웃 검색
3. 구현 전략 결정
    - 기존 컴포넌트 조합 우선
    - 새 컴포넌트는 재사용 불가한 경우에만 최소 단위로 작성
4. 1:1 구현
    - 색상, 타이포, 아이콘, spacing을 피그마 값 기준으로 적용
5. 편차 처리
    - 피그마에 없는 요소가 필요하면 임의 생성하지 않고 사용자 확인 요청

## 구현 체크리스트

-   기존 컴포넌트를 우선 사용했는가
-   동일 디자인 패턴을 프로젝트 내에서 재사용했는가
-   피그마 값과 불일치하는 컬러/폰트/아이콘이 없는가
-   피그마에 없는 요소를 임의로 추가하지 않았는가
